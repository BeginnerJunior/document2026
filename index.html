1. Tokens

The smallest unit an LLM processes. Text is split into tokens before processing.

  • A token is roughly ¾ of a word in English (e.g., "unhappiness" → un, happiness)
  • Models have a token limit that governs how much they can read and generate.
  • Cost/pricing is typically measured per token (input and output separately).
Think of tokens like puzzle pieces — the model reads and generates one piece at a time.

2. Context Window

The maximum number of tokens a model can handle in a single conversation (input + output combined).

ModelContext Window
GPT-4o~128K tokens
Claude Opus 4.6~1M tokens
Gemini 1.5 Pro~2M tokens
  • Larger context = can process more information at once (entire codebases, long documents).
  • Limitation: performance can degrade on information in the "middle" of very long contexts ("lost in the middle" problem).

3. Temperature

Controls randomness/creativity of the model's output.

ValueBehaviorUse Case
0.0Deterministic, most probable answerCode, math, factual Q&A
0.3 – 0.7Balanced creativityGeneral conversation, writing
1.0+Highly random, creativeBrainstorming, storytelling
  • Technically: temperature scales the probability distribution over the next token.
  • temperature=0 → always picks the highest-probability token.

4. Top-p (Nucleus Sampling)

  • An alternative to temperature for controlling randomness.
  • top_p=0.9 means: only consider tokens whose cumulative probability adds up to 90%.
  • Lower top-p = more focused; higher top-p = more diverse.
  • Often used together with temperature.

Quick Comparison

Precision «———————————» Creativity

temp=0          temp=0.5          temp=1.0
top_p=0.1       top_p=0.5         top_p=0.95

5. Prompting Techniques

Zero-Shot

Ask the model to do something with no examples. It relies entirely on pre-trained knowledge.

Classify this review as positive or negative:
"The food was terrible." → ?

One-Shot

Provide one example before the actual task.

Classify:
"Great service!" → Positive

"The food was terrible." → ?

Few-Shot

Provide a few examples (typically 2–5) to guide the model.

Classify:
"Great service!"    → Positive
"Awful experience." → Negative
"It was okay."      → Neutral

"The food was terrible." → ?
More examples = better pattern recognition, but uses more tokens.

Chain-of-Thought (CoT)

Instruct the model to reason step-by-step before giving the final answer.

Q: If a store has 23 apples and sells 17, how many are left?

A: Let's think step by step.
   - Start with 23 apples
   - Sold 17 apples
   - 23 - 17 = 6
   Answer: 6
  • Dramatically improves performance on math, logic, and multi-step reasoning.
  • Triggered by adding "Let's think step by step" to the prompt.

When to Use Which?

Simple task?            → Zero-shot     (just ask)
Need consistent format? → Few-shot      (give examples)
Complex reasoning?      → Chain-of-Thought (think step by step)
Very complex task?      → Break it into subtasks

8. How AI Understands Images & Video

The Core Idea: Multimodal Models

Traditional LLMs only understand text. Multimodal models can process text + images + audio + video by converting everything into the same representation.

Step 1: Vision Encoder

Images are split into small patches (like a grid), then each patch is converted into a numerical vector (embedding).

+---+---+---+---+
|   |   |   |   |
+---+---+---+---+   →  [0.23, 0.87, 0.12, ...]  (patch embeddings)
|   |   |   |   |   →  [0.45, 0.31, 0.98, ...]
+---+---+---+---+   →  [0.67, 0.54, 0.22, ...]
|   |   |   |   |   →  ...
+---+---+---+---+

Image patches  →  Vectors (same format as text tokens)

Common vision encoders: ViT CLIP SigLIP

Step 2: Projection Layer

Image embeddings are mapped into the same space as text token embeddings, so the LLM can "read" them.

Image patches → Vision Encoder → Projection → [image tokens]
                                                         |
Text input    → Tokenizer      →             [text tokens]
                                                         |
                                                  Combined sequence
                                                         |
                                                    LLM (Transformer)
                                                         |
                                                    Text output

Step 3: The LLM Processes Both Together

The model sees a single sequence of tokens — some from text, some from images. It uses the same attention mechanism on all of them.

  • Relate text to specific parts of the image
  • Answer questions about what it "sees"
  • Describe, analyze, or reason about visual content

How Video Works

Video is essentially a sequence of images (frames).

Video → Sample Key Frames → Encode Each Frame → Feed All Frame Tokens to LLM
  • Models sample frames at intervals (e.g., 1 per second) — not every frame is used.
  • Some models process audio tracks separately and combine them.
  • Video uses many more tokens than a single image.

Key Technologies

TechnologyRole
CLIP (OpenAI)Matches images with text descriptions — foundation for image understanding
ViTApplies Transformer architecture to image patches
Contrastive LearningTrains model to link "photo of a cat" with a cat image
Cross-AttentionLets text tokens "look at" image tokens and vice versa

Training Process

  1. Pre-training: Show millions of (image, text) pairs — model learns to connect visuals with words
  2. Instruction tuning: Train on Q&A about images ("What color is the car?" → "Red")
  3. RLHF: Human feedback to improve accuracy and safety
Images, text, audio, and video can all be represented as sequences of numerical vectors. Once everything is in the same format, a Transformer can process them all together.

Multimodal Summary

QuestionAnswer
How does AI "see"?Splits images into patches, converts to numerical vectors
How does it connect image to text?Both are mapped to the same vector space
How does video work?Sample frames → encode each → process as a long sequence
Why is this possible?Transformers can attend to any type of token in a sequence

9. How to Use AI Efficiently

Write Clear Prompts

The #1 factor in getting good results. Be specific about what you want.

Bad PromptGood Prompt
"Fix my code""Fix the null pointer error in the login function at line 42"
"Write about dogs""Write a 200-word paragraph about golden retrievers as family pets"
"Help me with SQL""Write a SQL query to find the top 10 customers by total order amount in 2025"

The CRAFT Framework

LetterMeaningExample
CContext"I'm a backend developer working on a REST API..."
RRole"Act as a senior code reviewer..."
AAction"Review this function and suggest improvements..."
FFormat"Return the result as a numbered list..."
TTone"Explain in simple terms for a junior developer..."

Choose the Right Temperature

TaskTemperatureWhy
Code generation0 – 0.2Need accuracy, not creativity
Data analysis0 – 0.3Factual, precise answers
General writing0.5 – 0.7Balance of quality and variety
Brainstorming0.8 – 1.0Want diverse, creative ideas

Break Complex Tasks into Steps

Don't ask AI to do everything at once. Decompose large tasks.

Instead of "Build me a complete e-commerce website", break it down:
  1. "Design the database schema for products, users, and orders"
  2. "Create the API endpoints for product CRUD operations"
  3. "Build the shopping cart logic"
  4. "Create the checkout flow"

Iterate, Don't Start Over

If the output isn't perfect, refine rather than rewrite.

You:  "Write a function to validate emails"
AI:   [gives basic regex]
You:  "Also handle edge cases like '+' aliases and subdomains"
AI:   [improved version]
You:  "Add unit tests for these cases"
AI:   [complete solution with tests]

Use System Prompts / Custom Instructions

Set persistent context so you don't repeat yourself.

System: You are a Python expert. Always use type hints.
        Follow PEP 8. Prefer list comprehensions over loops.
        Use pytest for testing.

Manage Token Usage (Save Cost & Speed)

StrategyHow
Be conciseRemove unnecessary words from prompts
Limit output"Answer in 3 bullet points" or set max_tokens
Use smaller models for simple tasksHaiku for classification, Opus for complex reasoning
Cache repeated contextUse prompt caching for repeated system prompts
Summarize long inputsCompress documents before asking questions

Model Selection Guide

Simple tasks (classify, extract, format)   → Haiku   (fast, cheap)
General tasks (write, explain, code)       → Sonnet  (balanced)
Complex tasks (architect, research, plan)  → Opus    (most capable)

Validate AI Output

AI can be confidently wrong (hallucination). Always verify:

  • Code: Run it. Write tests. Don't just trust it compiles.
  • Facts: Cross-check important claims with reliable sources.
  • Numbers: Recalculate critical math independently.
  • Links/URLs: AI often generates fake URLs. Check them.

Common Mistakes to Avoid

MistakeFix
Vague promptsBe specific about input, output, and format
Too much in one promptBreak into smaller tasks
Blindly trusting outputAlways review and test
Starting over on bad outputIterate and refine instead
Using the biggest model for everythingMatch model to task complexity
Not providing contextInclude relevant background info
Ignoring system promptsUse them to set consistent behavior

Efficiency Formula

Clear Prompt + Right Model + Right Technique + Right Temperature + Iteration + Validation
= Best Results at Lowest Cost
