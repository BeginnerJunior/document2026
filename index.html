Module 1

AI Under the Hood

How LLMs actually work — architecture, tokenization, context, and pricing.

What an LLM Actually Is

A Large Language Model (LLM) is a neural network that predicts the next token in a sequence. It does not think, reason, or understand — it performs sophisticated statistical pattern matching on training data. Output sounds intelligent because the training data was produced by intelligent humans, not because the model itself understands.

💡
Core insight: LLMs are next-token prediction engines. They sound confident even when completely wrong.

Architecture (Transformer)

  • Embedding Layer: Converts tokens to dense vectors (~12,288 dimensions for GPT-4). Adds positional encoding so word order is preserved.
  • Transformer Blocks: ~96 stacked layers, each refining token representations via self-attention.
  • Self-Attention: Each token computes Query (Q), Key (K), Value (V) vectors.
    Attention(Q,K,V) = softmax(Q·Kᵀ / √dₖ) · V
  • Multi-Head Attention: GPT-4 uses 128 attention heads per layer, each learning different relationship types.
  • Output Layer: Softmax converts raw scores into a probability distribution over ~100,000 vocabulary tokens.

Tokenization

LLMs read tokens, not words (using Byte Pair Encoding). Key implications:

  • Non-English text (Thai, Chinese, Japanese) uses 2–4× more tokens than English equivalents.
  • Numbers tokenize unpredictably — this is why LLMs cannot do arithmetic natively.
  • ~1 token ≈ 0.75 English words. A 50-page PDF ≈ 75,000 tokens.

Context Windows

The context window is the model's working memory. Beyond the limit, older content is silently dropped — without warning. Performance also degrades as the window fills; recall accuracy can drop 20–40% near the limit.

Module 2

Failure Modes

Where AI breaks down — and how to protect against it.

Attention Degradation — "Lost in the Middle"

Recall accuracy follows a U-shaped curve — high at the start and end of a document, dropping dramatically in the middle.

Position in DocumentRecall Accuracy
First 10%90%+
40–60% (middle)45–55% — DANGER ZONE
Last 10%85%+
⚠️
Mitigations: Put critical instructions at the start AND end; chunk long documents; use explicit section markers; repeat key constraints at the end.

Four Types of Hallucination

01

Factual Fabrication

Invented citations, nonexistent cases, fake statistics.
e.g., Mata v. Avianca 2023 — all 6 legal citations fabricated.

02

Confident Extrapolation

Real knowledge applied outside its valid domain.
e.g., US tax law applied to Thai questions.

03

Instruction Drift

Long outputs gradually deviate from original instructions.

04

Sycophantic Agreement

Model agrees with incorrect user premises rather than correcting them.

Risk Classification Framework

TierTypeExamplesGuardrail
1 — Low Risk AI generates, human scans Brainstorming, drafts, internal notes Minimal review
2 — Medium Risk AI assists, human verifies Policy summaries, customer comm drafts Mandatory human review before distribution
3 — High Risk Human decides, AI may inform Legal/financial decisions, hiring, regulatory submissions AI informs only; human makes ALL final decisions

Systematic Bias Sources

  • Training data bias — Overrepresents English, Western, and historical demographic norms.
  • Recency bias — Knowledge cutoff means outdated information may be stated confidently.
  • Western/English bias — Measurably lower quality output in Thai and other non-Latin-script languages.
  • Sycophancy bias — Tendency to validate user inputs, even incorrect ones.

🚫
Math Warning: ChatGPT, Claude, and Gemini in standard chat mode do NOT perform mathematical computation. Error rates on multi-digit multiplication exceed 40%. Always use Code Execution mode for any math or data task.

Retrieval-Augmented Generation (RAG)

RAG connects LLMs to private company data without fine-tuning:

  1. Ingest documents → Embed into vectors → Store in vector database
  2. Query by embedding user question → Retrieve top-K similar chunks
  3. Generate answer grounded in retrieved context
✅
RAG vs. Fine-Tuning: RAG is preferred for enterprise use — instant updates, source attribution, data stays on-premises, lower cost, and better hallucination control.

Model Selection Decision Tree

1
Math / counting / data? → Code Execution
2
Complex multi-step logic? → Reasoning model
3
Drafting / summarizing / brainstorming? → Fast non-reasoning model
4
Needs company-specific knowledge? → RAG or Custom Agent
5
Domain-specific (CRM, HR, BI)? → Specialized tool
Module 4

Prompt Engineering & Context Design

The RTCF framework and advanced prompting techniques.

The RTCF Framework

Every effective prompt needs four components:

ComponentWeak ExampleStrong Example
R — Role "Help me analyze this." "You are a senior financial analyst with 15 years in Thai consumer finance..."
T — Task "Tell me about Q3 performance." "Identify the top 3 cost centers that exceeded budget by >10%. Calculate variance, root cause, and Q4 corrective action."
C — Context (none) "Audience is the board. 5-minute attention window. Revenue +12%, OPEX +18%..."
F — Format (none) "Output: (1) 3-sentence exec summary. (2) Table: Cost Center, Budget, Actual, Variance. (3) Bullet recommendations by department."

Advanced Techniques

  • Chain-of-Thought (CoT): "Think step-by-step" improves logic/math accuracy by 20–40%.
  • XML Scratchpad: <scratchpad> for reasoning, <answer> for final output.
  • Few-Shot Examples: Provide 2–3 input/output pairs before your actual request.
  • Negative Prompting: "DO NOT use bullet points / add caveats / start with 'Certainly!'"
  • Structured JSON Output: Use response_format, tool use, or response_mime_type to guarantee valid JSON for downstream systems.

Context Hierarchy (Priority Order)

  1. System prompt (highest)
  2. Uploaded documents (high)
  3. Conversation history (medium)
  4. Current user message (variable)
  5. Training data (lowest — where hallucinations originate)

Anti-Patterns to Avoid

Anti-PatternDescription
Wall of TextPasting 20 pages with "summarize this"
Assumption of Knowledge"What about the Smith account?" — model has no idea

Premature Delegation"Should we approve this loan?" — model lacks your data

Data Classification Framework

LevelDefinitionAI Policy
PUBLIC Press releases, job postings, public filings Safe with any tool
INTERNAL Meeting notes, process docs, org charts Enterprise tools only
CONFIDENTIAL Financials, customer lists, salary data, contracts Explicit IT/DPO approval required
RESTRICTED Trade secrets, credentials, health records, board minutes, PII NEVER share with any AI system

The Retraining Risk

Pasting confidential data into a consumer AI → data enters training pipeline → future model may reproduce or reveal that information to anyone who asks a related question.

Documented Incidents

  • Samsung (2023): Engineers pasted proprietary source code into personal ChatGPT accounts.
  • Mata v. Avianca (2023): Lawyer submitted AI-fabricated legal citations; fined by court.
  • Italy (2023): ChatGPT temporarily banned by data protection authority over GDPR concerns.

Thailand PDPA Implications

  • Section 24: Personal data requires a valid legal basis to process.
  • Section 27: Consent must be explicit and specific — prior consent for loan processing does NOT cover AI tool processing.
  • Sections 77–90: Fines up to 5 million baht per violation; criminal liability possible.
⚖️
Accountability Standard: You are legally and professionally responsible for every output you send or publish. "The AI wrote it" is not a defense.

Pre-Send Checklist

  • Verified every fact, figure, citation independently
  • Reviewed for gendered/biased language
  • No confidential or restricted data revealed
  • Matches company tone and brand standards
  • No legal liability from claims that turn out to be wrong

1
"Would this matter if it appeared on the front page of a newspaper?" → If no: probably Public.
